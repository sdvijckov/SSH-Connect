# ⚠️ Решение проблем

## Ошибки подключения

### "Connection timed out"

**Возможные причины:**
- Неверный IP-адрес
- Проблемы с интернетом
- Брандмауэр блокирует

**Решение:**
Проверь адрес-сервера в письме от хостинга
```bash
# Проверь интернет-соединение
ping адрес-сервера

# Отключи брандмауэр на время
# Windows: Панель управления → Брандмауэр
```

---

### "Authentication failed"

**Причина:** Неверный пароль

**Решение:**
1. Проверь пароль в письме от хостинга
2. Убедись, что раскладка клавиатуры правильная (английская)
3. Попробуй скопировать пароль и вставить через **Shift+Insert**

---

### "Connection refused"

**Причина:** Неверный порт или SSH не запущен

**Решение:**
```bash
# Убедись, что порт 22
# Проверь, что SSH запущен на сервере
sudo systemctl status ssh

# Перезапусти SSH
sudo systemctl restart ssh
```

---

## Ошибки программы

### "ModuleNotFoundError: No module named 'paramiko'"

**Причина:** Не установлена библиотека paramiko

**Решение:**
```bash
pip install -r requirements_ssh.txt
```

---

### "ModuleNotFoundError: No module named 'cryptography'"

**Причина:** Не установлена библиотека cryptography

**Решение:**
```bash
pip install cryptography
```

---

### Программа не запускается

**Возможные причины:**
- Не установлен Python
- Не установлен Tkinter

**Решение:**
```bash
# Проверка Python
python --version

# Проверка Tkinter
python -c "import tkinter; print('OK')"

# Если Tkinter не установлен (Windows)
pip install tk
```

---

### "kitty.exe не найден"

**Причина:** Kitty не находится в папке проекта

**Решение:**
1. Скачай Kitty: https://www.9bis.net/kitty/
2. Помести `kitty.exe` в папку проекта:
   ```
   D:\Projects python\SSH Connect\kitty.exe
   ```

---

### "vncviewer.exe не найден"

**Причина:** VNC Viewer не находится в папке проекта

**Решение:**
1. Скачай VNC Viewer: https://www.realvnc.com/en/connect/download/viewer/
2. Помести `vncviewer.exe` в папку проекта

---

## Проблемы с Kitty

### Kitty не открывается автоматически

**Причины:**
- Файл `kitty.exe` отсутствует в папке проекта
- Неправильный путь к Kitty

**Решение:**
1. Убедись, что `kitty.exe` находится в папке:
   ```
   D:\Projects python\SSH Connect\kitty.exe
   ```
2. Проверь логи программы — там будет указан путь

---

### Kitty открывается и сразу закрывается

**Причины:**
- Неверные данные подключения
- Сервер отключает соединение

**Решение:**
1. Проверь данные подключения (IP, логин, пароль)
2. Попробуй подключиться вручную через Kitty
3. Проверь логи сервера

---

### Kitty открывается свёрнутым

**Это нормально!** Kitty специально открывается свёрнутым чтобы не мешать.

**Как развернуть:**
- Кликни на иконку Kitty в панели задач

## Проблемы с VNC

### "Too many authentication failures"

**Причина:** Зависшая сессия VNC

**Решение:**
```bash
# Перезапусти VNC
vncserver -kill :1
vncserver :1 -geometry 1280x800 -depth 24
```

---

### Серый экран вместо рабочего стола

**Причина:** Неправильный конфиг `xstartup`

**Решение:**
```bash
# Останови VNC
vncserver -kill :1

# Исправь конфиг
cat > ~/.vnc/xstartup << 'EOF'
#!/bin/bash
xrdb $HOME/.Xresources
startxfce4 &
EOF

chmod +x ~/.vnc/xstartup

# Запусти заново
vncserver :1 -geometry 1280x800 -depth 24
```

---

### VNC не запускается через кнопку "🖥️ Открыть GUI"

**Причины:**
- VNC сервер не установлен на сервере
- VNC сервер не запущен

**Решение:**
```bash
# Установи VNC (на сервере через Kitty)
sudo apt update
sudo apt install tightvncserver xfce4 -y

# Запусти VNC
vncserver :1

# Попробуй нажать "🖥️ Открыть GUI" снова
```

---

### VNC зависает

**Решение:**
1. В SSH Connect нажми **"🖥️ Открыть GUI"** — скрипт автоматически перезапустит VNC

Или вручную:
```bash
# Убить все сессии
vncserver -kill :1
vncserver -kill :2

# Запустить заново
vncserver :1 -geometry 1280x800 -depth 24
```

## Проблемы с файлами

### "Нет подключения"

**Решение:**
- Убедись, что подключился к серверу (кнопка "🔗 Подключиться")
- Проверь статус подключения (должен быть "🟢 Подключено")

---

### "Ошибка доступа"

**Причина:** Нет прав на файл

**Решение:**
```bash
# Проверь права доступа
ls -la filename

# Измени права
chmod 644 filename

# Или смени владельца
chown $USER:$USER filename
```

---

### "Не удаётся загрузить файл"

**Причины:**
- Нет места на сервере
- Не существует папка назначения

**Решение:**
```bash
# Проверь место
df -h

# Проверь папку
ls -la /home/sheldon/

# Создай папку если нет
mkdir -p /home/sheldon/
```

---

## Проблемы с шифрованием

### "Ошибка расшифровки пароля"

**Причина:** Утерян или повреждён файл `.encryption_key`

**Решение:**
1. Найди файл `.encryption_key` в папке проекта
2. Если файл утерян — пересоздай профили заново

**Важно:** Не удаляй файл `.encryption_key`!

---

### "Ошибка сохранения подключений"

**Причина:** Нет прав на запись в файл

**Решение:**
- Запусти программу от имени администратора
- Или проверь права на файл `ssh_connections.json`
