# 🔐 Безопасность и шифрование

## Шифрование паролей

SSH Connect использует шифрование для безопасного хранения паролей и SSH-ключей в файле `ssh_connections.json`.

### Как это работает

1. **При сохранении профиля:**
   - Пароль и SSH-ключ шифруются алгоритмом Fernet (AES-128-CBC)
   - Зашифрованные данные сохраняются в `ssh_connections.json`

2. **При загрузке профиля:**
   - Данные автоматически расшифровываются
   - Вы видите обычный пароль в поле ввода

3. **Ключ шифрования:**
   - Хранится в файле `.encryption_key` (в папке проекта)
   - Файл скрыт (атрибут hidden в Windows)
   - **Никогда не передавайте этот файл третьим лицам!**

---

## Файлы безопасности

| Файл | Описание | Можно передавать |
|------|----------|------------------|
| `ssh_connections.json` | Зашифрованные профили | ❌ Нет (содержит зашифрованные данные) |
| `.encryption_key` | Ключ шифрования | ❌ **КАТЕГОРИЧЕСКИ НЕТ** |
| `last_profile.txt` | Имя последнего профиля | ✅ Да (не содержит чувствительных данных) |

## Миграция на шифрование

Если у вас есть старые профили с незашифрованными паролями:

### Шаг 1: Установите cryptography

```bash
pip install -r requirements_ssh.txt
```

### Шаг 2: Запустите скрипт миграции

```bash
python migrate_encrypt.py
```

Скрипт:
- Проверит существующие пароли
- Зашифрует незашифрованные
- Сохранит обновлённый файл

## Важные предупреждения

### ⚠️ Не теряйте ключ шифрования!

Файл `.encryption_key` — это единственный ключ к вашим паролям.

**Если вы потеряете его:**
- ❌ Невозможно расшифровать пароли
- ❌ Придётся вводить все пароли заново
- ❌ Старые профили станут бесполезны

**Рекомендации:**
- Сделайте резервную копию `.encryption_key`
- Храните копию в безопасном месте
- Добавьте `.encryption_key` в резервное копирование

---

### ⚠️ Не передавайте файлы другим

**Никогда не передавайте:**
- `.encryption_key` — ключ шифрования
- `ssh_connections.json` — содержит зашифрованные пароли

**Можно передавать:**
- `last_profile.txt` — имя профиля
- Другие файлы проекта (кроме указанных выше)

## Технические детали

### Алгоритм шифрования

- **Алгоритм:** Fernet (симметричное шифрование)
- **Основа:** AES-128-CBC + HMAC
- **Ключ:** 128-битный (генерируется случайно)
- **Библиотека:** `cryptography` (Python)

### Формат зашифрованного пароля

Зашифрованный пароль выглядит так:
```
gAAAAABhZ2x5... (длинная строка)
```

**Структура:**
- `gAAAA` — префикс Fernet
- Остальная часть — зашифрованные данные + HMAC подпись

## Проверка шифрования

### Проверить статус шифрования

Откройте `ssh_connections.json` и посмотрите на поле `password`:

**Зашифровано ✅:**
```json
{
  "adminvps": {
    "password": "gAAAAABhZ2x5..."
  }
}
```

**НЕ зашифровано ❌:**
```json
{
  "adminvps": {
    "password": "MyPassword123"
  }
}
```

## Резервное копирование

### Что копировать

Для полного резервного копирования профилей:

1. `ssh_connections.json` — профили
2. `.encryption_key` — ключ шифрования
3. `last_profile.txt` — последний профиль (опционально)

### Что НЕ копировать

- `kitty.exe` — можно скачать заново
- `SSH_env/` — виртуальное окружение (можно создать заново)
- `__pycache__/` — кэш Python

## Вопросы и ответы

### Можно ли использовать один ключ на разных компьютерах?

**Да!** Скопируйте `.encryption_key` на другой компьютер вместе с `ssh_connections.json`.

### Как сменить ключ шифрования?

1. Удалите `.encryption_key`
2. Запустите программу — ключ создастся заново
3. Пересохраните все профили (пароли зашифруются новым ключом)

### Можно ли отключить шифрование?

**Не рекомендуется.** Но если нужно:

1. Откройте `src/utils.py`
2. В функциях `load_connections` и `save_connections` уберите вызовы `encrypt_field` и `decrypt_field`
